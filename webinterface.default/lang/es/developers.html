Información para desarrolladores

¿Quieres ayudar a hacer que Chorus sea aún mejor? Encuentra ayuda debajo ...

Esta página contiene información sobre cómo poner en marcha su entorno de desarrollo para que pueda construir y probar su cambios sin la molestia de configurar todas las dependencias requeridas.

Docker dev entorno

Incluido en este repositorio hay un Dockerfile que construye la imagen del entorno Chorus 2 dev. Si quisieras desarrollar sin uso de un docker, esta es una buena referencia en cuanto a lo que necesita instalar en su PC.

Si desea hacer su vida mucho más fácil, solo instale el docker y luego tome la imagen preconstruida del centro del docker hub.

docker pull jez500/chorus2-dev:latest

Instalar dependencias de desarrollo

Una vez que tenga la imagen de desarrollo del docker, puede usar esto para realizar todas las tareas relacionadas con el desarrollo. El primero de estos debe instalar todas las dependencias nodejs/ruby.

docker run -tiP -v `pwd`:/app jez500/chorus2-dev:latest ./build.sh install

Esto ejecutará npm install ybundle install dentro del contenedor dev.

Solo deberías hacer esto una vez, a menos que... package.json oGemfile sean actualizados

Construyendo/compilando el proyecto

Si ha realizado cambios en algún script de coffee o sass, puede crear esos cambios ejecutando comandos dentro de el contenedor dev. Para obtener una línea de comando en el contenedor:

docker run -tiP -v `pwd`:/app jez500/chorus2-dev:latest bash

Una vez dentro del contenedor dev, puedes hacer lo siguiente:

Construir

Esto construirá lenguajes, documentación, js y css.

grunt build

Esté atento a los cambios (construir continuamente)

Esto solo construirá js y css.

grunt

Remitiendo tus cambios

Como regla general, no debe remitir ningún archivo compilado a menos que esté creando una versión. P.ej. solo remitir archivos en la carpeta src
